## Why CrumbleCracker?

Containers share the host kernel. That's efficient, but it's also a security boundary with a history of breakouts. CrumbleCracker runs each workload in its own virtual machine with a dedicated kernel, while staying compatible with standard container tooling.

## See It In Action

```go
package main

import (
    "context"
    "fmt"
    cc "github.com/tinyrange/cc"
)

func main() {
    client, _ := cc.NewOCIClient()
    source, _ := client.Pull(context.Background(), "alpine:latest")

    // Boot a VM from a container image
    instance, _ := cc.New(source, cc.WithMemoryMB(256))
    defer instance.Close()

    // Run commands exactly like os/exec
    output, _ := instance.Command("cat", "/etc/os-release").Output()
    fmt.Println(string(output))
}
```
