## Why CrumbleCracker?

Containers share the host kernel. That's efficient, but it's also a security boundary with a history of breakouts. CrumbleCracker runs each workload in its own virtual machine with a dedicated kernel, while staying compatible with standard container tooling.

<FeatureGrid columns={4}>
  <FeatureCard title="Same Images, Real Isolation" icon="ðŸ³" color="blue">
    Pull from Docker Hub or any OCI registry. No conversion, no special builds. Your existing images work out of the box.
  </FeatureCard>
  <FeatureCard title="Hardware Security Boundaries" icon="ðŸ”’" color="purple">
    Each VM runs its own Linux kernel. An escape hits the hypervisor, not your host.
  </FeatureCard>
  <FeatureCard title="Familiar APIs" icon="âš¡" color="green">

    Hypervisor.framework on macOS, KVM on Linux, WHP on Windows. Uses each platform's native virtualization with no extra configuration.
  </FeatureCard>
</FeatureGrid>

## See It In Action

<LanguageTabs>
  <LangTabItem lang="go">
```go
package main

import (
    "context"
    "fmt"
    cc "github.com/tinyrange/cc"
)

func main() {
    client, _ := cc.NewOCIClient()
    source, _ := client.Pull(context.Background(), "alpine:latest")

    // Boot a VM from a container image
    instance, _ := cc.New(source, cc.WithMemoryMB(256))
    defer instance.Close()

    // Run commands exactly like os/exec
    output, _ := instance.Command("cat", "/etc/os-release").Output()
    fmt.Println(string(output))
}
```

  <LangTabItem lang="rust">
```rust
use cc::{OciClient, Instance, InstanceOptions};

fn main() -> cc::Result<()> {
    cc::init()?;

    let client = OciClient::new()?;
    let source = client.pull("alpine:latest", None, None)?;

    // Boot a VM from a container image
    let opts = InstanceOptions { memory_mb: 256, ..Default::default() };
    let instance = Instance::new(source, Some(opts))?;

    // Run commands with std::io patterns
    let output = instance.command("cat", &["/etc/os-release"])?.output()?;
    println!("{}", String::from_utf8_lossy(&output.stdout));

    cc::shutdown();
    Ok(())
}
```

import { init, shutdown, OCIClient } from '@crumblecracker/cc';

init();

const client = new OCIClient();
const source = await client.pull('alpine:latest');

// Boot a VM from a container image
await using instance = await source.createInstance({ memoryMb: 256 });

client.close();

  <LangTabItem lang="c">
```c
#include <libcc.h>
#include <stdio.h>

int main() {
    cc_error err = {0};
    cc_oci_client client;
    cc_instance_source source;
    cc_instance instance;

    cc_init();
    cc_oci_client_new(&client, &err);
    cc_oci_client_pull(client, "alpine:latest", NULL, NULL, NULL,
                       CC_HANDLE_INVALID(cc_cancel_token), &source, &err);

    // Boot a VM from a container image
    cc_instance_options opts = { .memory_mb = 256 };
    cc_instance_new(source, &opts, &instance, &err);

    // Run commands via handle-based API
    const char* args[] = {"/etc/os-release", NULL};
    cc_cmd cmd;
    cc_cmd_new(instance, "cat", args, &cmd, &err);
    uint8_t* output; size_t len; int exit_code;
    cc_cmd_output(cmd, &output, &len, &exit_code, &err);
    printf("%.*s", (int)len, output);

    cc_free_bytes(output);
    cc_instance_close(instance, &err);
    cc_shutdown();
    return 0;
}
```
